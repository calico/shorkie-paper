The 2A SpeciesLM row is regenerated once via the external HF model (`lm_SMT3_viz/0_compute_specieslm_smt3.py`, env `pytorch_cuda`, CPU); the 2A 15%-iterative row uses the **precomputed 3-architecture ensemble** (`preds_train.npz`, CPU, via `lm_SMT3_viz/6_extract_iterative_3arch.py`).

### Reproduction scope
- **Reproduced (CPU):** 2A (3 aligned SMT3 rows), 2D (the published 6-panel TSS-distance grid), 2E (t-SNE, all 16 chr).
- **Provided as upstream scripts, not re-rendered:** 2C — the 6-dataset × 11-motif TF-MoDISco conservation grid is generated by the original author scripts `motif_lm/4_viz_motif.py` + `motif_lm__unseen_species/4_viz_motif.py` (refactored to config-driven paths, with `.sh` wrappers). It depends on the full per-tier MoDISco `.h5` set, so the notebook points to those scripts rather than re-deriving the grid (same treatment as the Fig 5 motif-progression panels).
- **Out of scope:** 2B — the per-iteration 15%-masked prediction matrix (a GPU forward pass) is not part of this reproduction.
