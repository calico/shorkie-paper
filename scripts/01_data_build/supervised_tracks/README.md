# Supervised-track data build

> **Advanced / reproduction-only.** You don't need this to *run* Shorkie — download released
> artifacts with `data/download.sh` and use `examples/` + `minimal_example/`. This stage rebuilds
> the training data; you only need it to fine-tune Shorkie on your own RNA-seq tracks.

Produces the **5,215-track**, 8-fold TFRecord dataset that Shorkie
(`shorkie_finetuned` and `shorkie_scratch`) trains on: raw FASTQ → aligned BAM →
coverage BigWig → (via `hound_data.py`) TFRecords on the *S. cerevisiae* R64
(S288C R64-3-1) genome at 16 bp resolution.

> **Practical entry point:** the processed BigWigs (~93 GB) and TFRecords are
> released — `data/download.sh --supervised`. Rebuild from raw FASTQ only if you
> need to reproduce the tracks themselves; the raw sequencing is large and uses
> several specialized conda envs.

## Pipeline

```
FASTQ ──(align)──► BAM ──(coverage)──► BigWig ──(make_data.sh / hound_data.py)──► TFRecords
```

| Stage | Script | Conda env | Tools |
|---|---|---|---|
| ChIP-exo align | `chipseq2bam.sh`, `process_ChIPs.py` | (tools on PATH) | `bwa mem`, `samtools`, Picard `MarkDuplicates`, `bedtools` |
| RNA-seq align | `rnaseq2bam.sh`, `process_RNAseq.py` | `idea_rnaseq` | `STAR` (`quantMode TranscriptomeSAM`, introns 10–2500), Picard |
| coverage → BigWig | `coverage.sh`, (`bam_cov.py`) | `basenji` | `bedtools genomecov`, `bedGraphToBigWig`, `bam_cov.py` |
| MNase coverage | `process_MNase.sh` | `yeast_seq` | Picard, `bam_cov.py` |
| peak calling | `run_macs.sh` | `macs` | `MACS3 callpeak` (`--nomodel --extsize 16 -q 0.05`, gsize 12,071,326), `bdgcmp` logFE |
| **TFRecord build** | **`make_data.sh`** → `hound_data.py` | `yeast_ml` | `baskerville` |

BigWig naming convention (the `file` column of the targets sheet points at these):
`*_coverage.bw` (basic genomecov), `*_bamcov.bw` (`bam_cov.py`), `*_macs.bw` /
`*_pos_logFE.bw` / `*_peaks_logFE.bw` (MACS logFE).

## `make_data.sh` — TFRecord build

Wraps the exact released `hound_data.py` invocation, parameterized through
`config/paths.yaml`:

```bash
scripts/01_data_build/supervised_tracks/make_data.sh --dry-run   # print the command
scripts/01_data_build/supervised_tracks/make_data.sh             # run it
```

```
hound_data.py --local -p 200 -l 16384 -w 16 -c 1024 -f 8 -r 256 --stride 6165 \
  -b <mask_rossi.bed> -u <mask_rossi.bed> --umap_clip 0.5 -g <assembly_gaps.bed> \
  -o data  <S288C_R64-3-1.fasta>  <cleaned_sheet.txt>
```

Flags: `-l 16384` seq length · `-w 16` bin width · `-c 1024` end crop · `-f 8`
folds · `-r 256` seqs/TFRecord · `--stride 6165` train stride · `-b/-u` blacklist
+ unmappable (both `mask_rossi.bed`) · `--umap_clip 0.5` · `-g` assembly gaps.

The genome FASTA is the SGD **S288C R64-3-1** assembly (config key
`genome.r64_supervised_fasta`; not committed — ~12 MB; obtainable from SGD /
Ensembl). The blacklist/gaps BEDs are the committed `data/R64_annotations/`
copies.

## Targets sheet (`cleaned_sheet.txt`)

Tab-separated; one row per track. `wc -l` = 5,216 (1 header + 5,215 tracks).
Config key: `datasets.targets_sheet`.

| Column | Meaning |
|---|---|
| `index` | 0-based track index |
| `identifier` | track ID (e.g. `AAP1_S0`, `ARG80_T0_S757`, `ERR9593592`) |
| `file` | absolute path to the track's BigWig — **rewrite to your local BigWig dir** before building |
| `sum_stat` | pooling statistic (`sum`) |
| `clip_soft` | soft-clip threshold (100000 for all tracks) |
| `description` | human-readable (assay + coverage) |
| `group` | source group (below) |

Track groups (`group` column):

| group | # tracks | source |
|---|---|---|
| `Other` | 1,128 | Rossi ChIP-exo |
| `RNA-Seq` | 3,053 | IDEA induction RNA-seq |
| `1000-RNA-Seq` | 1,014 | yeast-strain RNA-seq (ENA `ERR959*`) |
| `Chip-MNase` | 20 | histone-mark MNase |

## Raw data provenance

Primary references (from the top-level `README.md`, "Training Data
Availability"). The processed BigWigs are the released entry point; raw-FASTQ
accessions should be confirmed against the paper's Methods before redistribution.

- **IDEA induction RNA-seq** (`RNA-Seq` group) — Induction Dynamics gene
  Expression Atlas; new data generated by Calico (related to IDEA 1.0). Hackett,
  S.R. *et al.*, *Mol Syst Biol* (2020).
- **Yeast-strain RNA-seq** (`1000-RNA-Seq` group) — diverse *S. cerevisiae*
  strains; ENA run accessions `ERR959*`. Caudal, É. *et al.*, *Nat Genet* (2024).
- **ChIP-exo & ChIP-MNase** (`Other`, `Chip-MNase` groups) — Rossi, M.J. *et al.*,
  *Nature* (2021).
